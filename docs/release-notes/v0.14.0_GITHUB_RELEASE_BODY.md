# Sona v0.14.0 - Developer Usability Release

Sona `0.14.0` is a production-quality developer usability release. It focuses
on first-run clarity: cleaner CLI behavior, better diagnostics, validated
examples, clearer docs, leaner Python packages, and improved VS Code extension
presentation.

This release does not add new syntax, stdlib APIs, governance architecture, or
major runtime behavior. It removes friction from the existing 0.13.x foundation.

## Highlights

- Cleaner CLI first-run flow:
  - `sona --version`
  - `sona --help`
  - `sona hello.sona`
  - `sona run hello.sona`
- Better edge-case behavior for common beginner paths:
  - `sona` with no args prints help and exits nonzero.
  - `sona run` with no file reports `SonaUsageError`.
  - `sona --unknown` reports `SonaUsageError`.
  - missing files report `SonaFileError`.
- Canonical diagnostics guide:
  - `docs/errors/v0.14-diagnostics.md`
  - compact `SonaXError: message` output
  - one-line actionable hints
  - `--errors=explain|trace|both`
- Improved import typo diagnostics:
  - `SonaImportError: module 'mathh' not found`
  - `hint: did you mean 'math'?`
- New and tightened docs:
  - `docs/QUICKSTART.md`
  - `docs/LANGUAGE_REFERENCE.md`
  - `docs/STDLIB_REFERENCE.md`
  - historical `docs/errors/v0.10-errors.md` now points to the v0.14 guide
- Official examples are validated from source checkout via:
  - `python tools/run_examples.py`
- Python package stays lean:
  - examples/docs/tests are not required inside installed packages
- VS Code extension README now includes a one-screen "How to use Sona in VS Code" section.

## First Run

Install:

```bash
pip install sona-lang
sona --version
sona --help
```

Bash or macOS/Linux shell:

```bash
echo 'print("Hello, Sona!")' > hello.sona
sona hello.sona
```

Windows PowerShell:

```powershell
'print("Hello, Sona!")' | Out-File -Encoding utf8 hello.sona
sona hello.sona
```

## Source-Checkout Examples

Official examples are part of the repository validation surface, not the
installed package payload.

Run one example:

```bash
sona run examples/hello.sona
```

Run the full official example suite:

```bash
python tools/run_examples.py
```

Expected runner shape:

```text
Running 9 examples...
OK hello.sona
...
All examples passed.
```
