# Sona v0.13.0

Sona v0.13.0 is a persistent memory governance release. It adds deterministic policy enforcement, append-only audit logging, and read-only compliance reporting to the runtime memory subsystem — keeping the language surface stable while hardening the runtime for production traceability.

## Highlights

- **Policy Engine** — `PolicyKernel` with `MemoryPolicy` and 5 sub-policies (classification, retention, capacity, access, governance) enforcing constraints at the storage boundary.
- **Audit Log** — `AuditKernel` with self-bootstrapping `audit_log` SQLite table, forensic export, and episode immutability verification.
- **Compliance Reporter** — `ComplianceReporter` scanning episodes, goals, checkpoints, and audit events against policy constraints, producing structured `ComplianceReport` with violation details.
- **Interpreter Integration** — `AuditKernel` and `ComplianceReporter` created and wired in the interpreter constructor. Audit logging active in intake, inspection, and goal-loop paths.
- **Advanced Exports** — All governance types exported from `sona.runtime.memory.advanced`.

## Notes

- No breaking changes to the public language surface.
- This is a governance and trust-infrastructure release, not a feature-expansion release.
- All governance modules are backward-compatible — audit wiring uses optional `None` defaults.
- Root `vscode-extension/` wrapper package still carries `axios`; do not publish that wrapper artifact as part of this release.
