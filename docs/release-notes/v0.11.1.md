# Sona v0.11.1 - Minor Stabilization Update

**Release tag:** `v0.11.1`  
**Date:** 2026-03-14  
**Status:** Draft

Sona v0.11.1 is a conservative stabilization release. It keeps scope intentionally narrow: ship a small set of safe, validated language/runtime improvements, demonstrate forward motion, and avoid pulling larger cognitive-runtime architecture changes into a patch line too early.

## Included in 0.11.1

### Control-flow fixes

- Fixed `when` subject matching so `when x { 5 => ... }` compares case values against the evaluated subject instead of treating each case as an independent truthy condition.
- Removed dead code after `raise ContinueException()` in `ContinueStatement`.
- Removed ghost `control_flow_integration` checks from `EnhancedIfStatement` and `EnhancedWhileLoop`.

### Small language enhancement

- Added for-loop destructuring support for `for a, b in value { ... }`.
- Lists destructure as `(index, value)`.
- Dictionaries destructure as `(key, value)`.

### Runtime performance

- Reused a single `SonaParserv090` instance inside `SonaUnifiedInterpreter` instead of rebuilding the Earley parser on every `_execute_sona_code()` call.
- This removes repeated parser construction during startup and stdlib module loading.

### Cleanup and alignment

- Removed orphaned ternary-expression AST/transformer code left behind after the grammar experiment was reverted.
- Aligned version metadata and REPL/runtime banners to `0.11.1`.

## Explicitly out of scope

- No first-class cognitive runtime, persistent memory system, belief graph, replay engine, or agent-runtime substrate ships in `0.11.1`.
- No major syntax expansion beyond the validated destructuring addition ships in this release.
- No risky architectural rewrite is included just to signal roadmap progress.

## Why this release is small

- The control-flow and parser reuse fixes are already implemented and validated.
- They improve correctness and startup behavior without changing Sona's overall architecture.
- This keeps `0.11.1` reasonable, easy to explain, and low risk to ship.

## Notes

- This release should remain narrowly scoped to the control-flow fixes, parser reuse, for-loop destructuring, and version/documentation alignment.
- VS Code extension cleanup warnings are not considered patch blockers by default, but dependency advisories should be reviewed before publishing extension artifacts.

## Forward path

The larger receipts, traces, replay, belief-graph, and cognitive-operating-system direction remains a post-`0.11.1` design track. `0.11.1` should only signal disciplined progress, not attempt to land that whole architecture early.
