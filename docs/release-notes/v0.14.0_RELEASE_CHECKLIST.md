# Sona v0.14.0 Release Checklist

## CLI

- [x] `sona --version` prints one line: `Sona 0.14.0`.
- [x] `sona --help` is grouped and human-readable.
- [x] `sona` with no arguments prints help to stderr and exits `1`.
- [x] `sona run` with no file reports `SonaUsageError` and exits nonzero.
- [x] `sona --unknown` reports `SonaUsageError` and exits nonzero.
- [x] `sona run examples/hello.sona` works.
- [x] `sona examples/hello.sona` works.
- [x] Missing-file errors print to stderr and exit nonzero.
- [x] Import typos use `SonaImportError` and obvious stdlib suggestions.

## Docs

- [x] `docs/QUICKSTART.md` exists and matches runtime behavior.
- [x] `docs/LANGUAGE_REFERENCE.md` documents stable behavior only.
- [x] `docs/STDLIB_REFERENCE.md` documents the stable user-facing stdlib surface.
- [x] `docs/errors/v0.14-diagnostics.md` documents only runtime-backed error behavior.
- [x] `docs/errors/v0.10-errors.md` is marked historical and links to v0.14 diagnostics.
- [x] Runnable docs examples pass.

## Examples

- [x] Official examples exist in the source repository.
- [x] Examples are validated from the source repository only.
- [x] Official examples exist with stable filenames.
- [x] `python tools/run_examples.py` passes.
- [x] Examples do not require external services or persistent memory state.
- [x] Examples are not required to exist in installed packages.

- [x] `python -m pytest -o addopts='' tests/test_examples_run.py tests/test_stdlib_docs_examples.py -q --no-cov`
- [x] `python -m pytest -o addopts='' tests/runtime -q --no-cov`
- [x] `python -m pytest -o addopts='' tests/test_smod_metadata_contracts.py -q --no-cov`
- [x] `python -m pytest -o addopts='' tests/test_smod_native_modules.py -q --no-cov`

Known limitation: the runtime test suite is slow and can require more than 10
minutes. This is not a release blocker for `0.14.0` if split-file runtime gates
pass.
