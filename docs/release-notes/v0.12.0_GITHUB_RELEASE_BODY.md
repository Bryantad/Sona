# Sona v0.12.0

Sona v0.12.0 is the frozen language-first boundary and packaging cut that was finalized before the repo moved on to the 0.13.x line. The language surface stays stable, while the runtime-memory public boundary is tightened, release metadata is brought into sync, and the ship artifacts for this cut are validated end to end.

## What Was Frozen Into 0.12.0

- Reduced `sona.runtime.memory` to the stable public model layer.
- Split host-oriented runtime surfaces into `sona.runtime.memory.advanced` and `sona.runtime.memory.internal`.
- Updated first-party interpreter and runtime imports to respect the narrower public boundary.
- Hardened goal continuation so continuation cannot proceed without a complete deterministic explanation envelope.
- Added the low-risk nested stdlib helper `stdlib/utils/convert.smod`.
- Aligned core package metadata, stdlib metadata, and the nested VS Code extension package to `0.12.0`.
- Drafted and locked the release documentation for this cut under `docs/release-notes/`.

## Notes

- `0.12.0` is a boundary and packaging release, not a broad runtime-platform redesign.
- No large native stdlib migration sweep ships in this cut beyond the targeted nested helper addition.
- Root `vscode-extension/` wrapper package still carries `axios`; do not publish that wrapper artifact as part of `0.12.0`.
- No breaking language changes are intended for existing users in this release.
- Public PyPI publishing uses `sona-lang` because the `sona` project name is already owned on PyPI.
- The published PyPI package page should describe the `0.12.0` release line rather than the old `0.10.1` release page.
