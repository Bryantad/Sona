# Sona v0.10.3 (Cumulative Since v0.10.1)

Release date: February 19, 2026

This release body summarizes all user-facing changes delivered after `v0.10.1`, covering `v0.10.2` and `v0.10.3`.

## Summary

From `v0.10.1` to `v0.10.3`, Sona moved from stability/alignment into two concrete upgrades:

- parser/runtime blocker closure (`v0.10.2`)
- native `.smod` runtime milestone expansion (`v0.10.3`)

## Changes in v0.10.2

- Fixed `repeat while` and `repeat until` runtime execution.
- Fixed triple-quoted multiline string parsing/execution.
- Added `.smod` support for `string.template(...)`.
- Fixed `test.only(...)` and `assert.throws(...)` interoperability with Sona functions.
- Fixed LSP outline symbol positioning for `func` and `def`.
- Added `scripts/release_gate.ps1` for repeatable local release checks.

## Changes in v0.10.3

- Added final hard native module: `stdlib/regex.smod` (bridge-free runtime path).
- Added native `.smod` module: `stdlib/uuid.smod`.
- Added safe digest module: `stdlib/hashing.smod` (bridge-backed for crypto correctness).
- Confirmed native must-have set stability:
  - `queue`
  - `stack`
  - `random`
  - `uuid`
  - `regex`
  - `hashing`
- Added explicit `.smod` runtime contract metadata:
  - `module_format = "smod-runtime"`
  - `runtime_backend = "smod"` or `runtime_backend = "smod-bridge"`
- Updated packaging metadata to include top-level `stdlib/*.smod` files in distribution artifacts.

## Validation Snapshot Across 0.10.2 and 0.10.3

- `v0.10.2`:
  - blocker scenarios passed (`repeat`, multiline strings, stdlib additions, test extras)
  - `python -m pytest -q tests -x` passed
- `v0.10.3`:
  - `python -m pytest tests/test_smod_native_modules.py -q` -> `6 passed`
  - `python run_sona.py test_all_30_imports.sona` -> `30/30 imports successful`
  - `python run_sona.py test_stdlib_regex.sona` -> complete (no runtime crash)

## Upgrade Notes (v0.10.1 -> v0.10.3)

- No intentional breaking changes were introduced in this range.
- `regex.smod` is intentionally regex-lite for native runtime stability.
- `hashing.smod` intentionally uses a bridge-backed backend for cryptographic correctness.
