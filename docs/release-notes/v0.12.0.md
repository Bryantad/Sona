# Sona v0.12.0 - Language-First Boundary Release

**Release tag:** `v0.12.0`  
**Date:** 2026-03-16  
**Status:** Published

Sona v0.12.0 is a packaging and architecture-signal release. It keeps the language surface stable while tightening the public runtime boundary, adds one low-risk native `.smod` improvement, and aligns core package, stdlib, and VS Code extension metadata for the next release line.

## Included in 0.12.0

### Runtime boundary cleanup

- Reduced `sona.runtime.memory` to the stable public model layer.
- Moved host/runtime kernels and storage behind `sona.runtime.memory.advanced`.
- Moved goal/checkpoint/orchestration helpers behind `sona.runtime.memory.internal`.
- Updated first-party interpreter/runtime imports to use the explicit surfaces.

### Language-first architecture signal

- Added runtime architecture documentation clarifying that the cognitive/runtime stack remains subordinate to the language.
- Corrected the main export surface that most strongly suggested framework drift.

### Quick native `.smod` win

- Added `stdlib/utils/convert.smod` as a low-risk nested stdlib helper.
- Kept the native-module scope intentionally narrow for this release.

### Packaging and metadata alignment

- Bumped core package metadata to `0.12.0`.
- Brought stdlib metadata forward from `0.11.0` to `0.12.0`.
- Updated VS Code extension manifests, badges, bundled runtime version, and changelogs.
- Public PyPI publication for this release uses the distribution name `sona-lang`.
- The PyPI package page description was refreshed so the published `0.12.0` package no longer presents stale `v0.10.1` copy.

## Explicitly out of scope

- No large cognitive-runtime redesign ships in `0.12.0`.
- No broad native stdlib migration sweep is included beyond quick wins.
- No repo-wide lint cleanup is included just because diagnostics exist.
