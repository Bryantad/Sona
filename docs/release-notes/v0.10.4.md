# Sona v0.10.4 - Trust-Layer Reliability Baseline

> **⚠ Absorbed into v0.11.0.** This version was consolidated into the v0.11.0 release. No separate v0.10.4 tag will be published. See [v0.11.0.md](v0.11.0.md) for the complete record.

**Release tag:** `v0.10.4` → merged into `v0.11.0`  
**Date:** 2026-02-24 (original) → 2026-03-03 (as v0.11.0)

Sona v0.10.4 locks the governance baseline for the trust-layer roadmap: deterministic receipt anchors, chain verification tooling, restored control-plane commands, and cross-OS trust smoke gates.

## Scope

- This release is intentionally scoped to reliability and governance baselines.
- v0.10.5 items remain out of scope for this tag:
  - receipt signing/verify-signature
  - receipt redaction/encryption
  - receipt indexing/query service
  - policy static analyzer and tool boundary governor expansion

## Highlights

- Added trust-layer CI smoke matrix (`Windows`, `Linux`, `macOS`) in `.github/workflows/ci.yml`.
- Added `sona receipt verify-chain` for chain integrity checks.
- Restored control-plane modules:
  - `sona.policy`
  - `sona.lockfile_manager`
- Receipt model now includes governance anchors:
  - `header.policy_fingerprint`
  - `header.engine_policy_fingerprint`
  - `header.prev_receipt_hash` (optional)
  - canonical `receipt_hash`
- Wave 1 metadata contracts explicitly added for `.smod` modules:
  - `json`, `fs`, `io`, `path`, `env`

## Governance and Receipts

### Receipt chain baseline

- `sona receipt scan/build` supports:
  - `--prev-receipt`
  - `--prev-hash`
- `sona receipt verify-chain` validates:
  - embedded hash vs canonical hash
  - `prev_receipt_hash` linkage
  - first divergence index/reason
- Chain verification emits summary certificate fields:
  - chain length
  - head/tail hash
  - disconnected receipt count

### Policy and lock reliability

- `sona probe` now reports policy snapshot and policy fingerprint.
- `sona lock` and `sona verify` are restored through `sona.lockfile_manager`.
