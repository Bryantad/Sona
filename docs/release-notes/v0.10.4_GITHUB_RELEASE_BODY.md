# Sona v0.10.4

Release date: February 24, 2026

Sona v0.10.4 locks the trust-layer reliability baseline: governance anchors in receipts, receipt hash-chain verification tooling, restored policy/lock control-plane commands, and cross-OS trust smoke gates.

## Headline Changes

- Added `sona receipt verify-chain` (chain integrity at rest).
- Receipt headers now carry:
  - `policy_fingerprint`
  - `engine_policy_fingerprint`
  - `prev_receipt_hash` (optional)
  - canonical `receipt_hash`
- Restored:
  - `sona.policy` support (`sona probe` operational)
  - `sona.lockfile_manager` support (`sona lock` / `sona verify` operational)
- Added `trust-layer-smoke` matrix CI (`Windows`/`Linux`/`macOS`).
- Published draft baselines:
  - `docs/spec/DETERMINISM_DRAFT_1.md`
  - `docs/spec/RECEIPT_SPEC_0.3_DRAFT.md`

## Native `.smod` Contract Progress

- Explicit metadata added for Wave 1 modules:
  - `json`, `fs`, `io`, `path`, `env`
- Top-level `.smod` contract coverage:
  - `11/16` explicit metadata contracts
  - `5` modules remaining in backlog (`csv`, `date`, `math`, `string`, `time`)

## Scope Guard (What is not in 0.10.4)

Deferred to v0.10.5+:

- receipt signing and signature verification APIs
- receipt redaction/encryption
- receipt index/query layer
- policy static analyzer and extended tool-boundary governance
