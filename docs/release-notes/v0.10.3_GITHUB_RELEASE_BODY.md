# Sona v0.10.3

Release date: February 19, 2026

Sona v0.10.3 completes the native `.smod` milestone by adding the final hard module (`regex`), plus a safe `hashing` `.smod` module for cryptographic digests.

## Highlights

- Final hard native module added: `stdlib/regex.smod`
- Native must-have set now includes:
  - `queue`
  - `stack`
  - `random`
  - `uuid`
  - `regex`
  - `hashing`
- Native module contract is explicit:
  - `module_format = "smod-runtime"`
  - `runtime_backend = "smod"` (bridge-free modules)
  - `runtime_backend = "smod-bridge"` (hashing safety/correctness path)
- Packaging now includes top-level native `stdlib/*.smod` files in distribution artifacts.

## What Changed

### Native runtime modules

- `regex` migrated to a bridge-free `.smod` implementation:
  - `compile`, `match`, `fullmatch`, `search`, `test`
  - `find_all` / `findall`
  - `replace`, `split`, `escape`
- Existing native `.smod` modules retained and validated:
  - `queue`, `stack`, `random`, `uuid`
- Added `hashing` module entrypoint as `.smod` with stable digest surface:
  - `sha256`, `hash`, `digest`, `checksum`
  - Compatibility helpers: `md5`, `sha1`, `sha512`, `sha3_256`, `sha3_512`, `blake2b`, `hmac_sha256`
  - Uses `runtime_backend = "smod-bridge"` for cryptographic correctness via native digest functions.

### Packaging/release hardening

- Included top-level native module files in package metadata so installed runtimes can resolve `.smod` modules:
  - `MANIFEST.in`
  - `pyproject.toml`
  - `setup.py`
- Release gate defaults updated for `0.10.3`:
  - `scripts/release_gate.ps1`

## Notes

- `regex.smod` is intentionally regex-lite and optimized for native `.smod` runtime stability rather than full Python `re` parity.
- `hashing.smod` is intentionally safety-first and bridge-backed for stable cryptographic correctness.
- v0.10.3 scope is native `.smod` migration and release packaging correctness.
