# Sona v0.10.3 - Native `.smod` Module Milestone

**Release tag:** `v0.10.3`  
**Date:** 2026-02-19

Sona v0.10.3 extends the native `.smod` runtime migration by adding one more must-have module and hardening the migration validation path.

## Highlights

- Added `regex` as the final hard native `.smod` module (`stdlib/regex.smod`) with no `__native__` wrapper dependency.
- Added `uuid` as a pure native `.smod` runtime module (`stdlib/uuid.smod`).
- Added `hashing` as a safe `.smod` digest module (`stdlib/hashing.smod`) backed by native digest functions.
- Confirmed prior native `.smod` modules remain stable:
  - `queue`
  - `stack`
  - `random`
  - `uuid`
  - `hashing`
- Expanded focused migration tests to cover `uuid` and `regex` behavior plus metadata contract.
- Packaging metadata now includes top-level `stdlib/*.smod` files in release artifacts.

## Native Module Contract

Migrated modules expose:

- `module_format = "smod-runtime"`
- `runtime_backend = "smod"` (bridge-free modules)
- `runtime_backend = "smod-bridge"` (safety/correctness bridge-backed modules)

This confirms `.smod` runtime execution with explicit backend mode per module (bridge-free where possible, bridge-backed where needed for correctness).

## UUID Module Surface

`stdlib/uuid.smod` now provides:

- UUID generation helpers: `v4`, `uuid4`, `v1`, `uuid1`
- Deterministic namespace variants: `v3`, `v5`, `uuid5`
- Validation and utility: `is_valid`, `nil`, `generate`, `short`
- Conversion helpers: `to_bytes`, `from_bytes`
- Namespace constants:
  - `NAMESPACE_DNS`
  - `NAMESPACE_URL`
  - `NAMESPACE_OID`
  - `NAMESPACE_X500`

## Regex Module Surface (final hard native module)

`stdlib/regex.smod` now runs as a bridge-free native module and provides:

- `compile`
- `match`
- `fullmatch`
- `search`
- `test`
- `find_all` / `findall`
- `replace`
- `split`
- `escape`

Notes:

- The current implementation is regex-lite and prioritizes native `.smod` runtime stability over full Python `re` parity.
- This module now executes natively in `.smod` without `__native__` bridge calls.

## Hashing Module Surface (safe digest module)

`stdlib/hashing.smod` now provides:

- `sha256`, `hash`, `digest`, `checksum`
- compatibility helpers: `md5`, `sha1`, `sha512`, `sha3_256`, `sha3_512`, `blake2b`, `hmac_sha256`

Notes:

- `hashing.smod` uses `runtime_backend = "smod-bridge"` to preserve cryptographic correctness and compatibility.
- SHA-256 outputs match standard vectors in focused tests.
