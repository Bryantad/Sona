# Sona v0.11.1 Plan - LSP Reliability Hotfix

**Status:** Draft  
**Target window:** Next patch release after `v0.11.0`

## Release Goal

Stabilize VS Code language server startup so missing local dependencies do not cause crash loops, noisy logs, or repeated failed restarts.

## Scope (In)

- LSP dependency preflight before startup (`pygls` import check on selected Python).
- One-time blocking behavior when preflight fails (no repeated start attempts in same VS Code session).
- Clear actionable remediation in UX:
  - show error once
  - offer copyable install command
  - direct users to `Sona LSP` output channel
- Reduce non-actionable output spam from non-`.sona` documents.
- Keep `.sona`-only lazy start behavior.

## Scope (Out)

- Language server protocol feature additions.
- Semantic tokens/completion/diagnostics feature expansion.
- Major extension architecture changes.

## Acceptance Criteria

- Opening non-`.sona` files does not flood output with `maybeStart` noise.
- Opening a `.sona` file with missing `pygls`:
  - shows one clear setup error
  - does not enter restart crash loop
  - does not repeatedly prompt on every editor event
- Opening a `.sona` file with `pygls` installed starts LSP successfully.
- `node --check` passes for packaged extension runtime files.

## QA Matrix

- Windows + Python in PATH
- Windows + workspace `.venv`
- macOS/Linux + workspace `.venv`
- Missing `pygls` vs installed `pygls`

## Candidate Changelog Entries

- Fixed VS Code LSP restart loop when `pygls` is missing.
- Added LSP dependency preflight and one-time recovery prompt.
- Reduced noisy non-`.sona` activation logs in extension output.
