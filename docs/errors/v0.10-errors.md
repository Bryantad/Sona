# Sona v0.10.0 - Historical Error Behavior

This page is historical. For the current Sona `0.15.1` user-facing diagnostics
contract, use [`v0.14-diagnostics.md`](v0.14-diagnostics.md).

v0.10.0 is the baseline for deterministic error reporting.

## What you can rely on

- Syntax errors stop execution.
- Runtime errors stop execution unless caught with `try/catch`.
- `run_sona.py` prints a human-readable message; `--debug` includes a Python traceback.

## Common runtime errors

- Undefined variable: `NameError`
- Invalid import: `ImportError`
- Invalid index/key: typically `IndexError` or `KeyError`
- Wrong argument count: `TypeError`

## Notes for v1.0

A unified error printer with `file:line:col` and error codes was a v1.0 readiness
item in the v0.10 release line.
